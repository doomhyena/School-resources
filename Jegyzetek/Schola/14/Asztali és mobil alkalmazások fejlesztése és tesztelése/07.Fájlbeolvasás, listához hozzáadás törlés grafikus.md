## 2) Osztály és a két lista modellje

```java
public class Main {
    static DefaultListModel<Eloado> eloadok = new DefaultListModel<>();
    static JList<Eloado> lbZenek = new JList<>(eloadok);

    static DefaultListModel<Eloado> kedvencEloadok = new DefaultListModel<>();
    static JList<Eloado> lbKedvencek = new JList<>(kedvencEloadok);
```

Itt 2 fontos fogalom van:

### a) `DefaultListModel<Eloado>`

Ez tárolja a lista elemeit (mint egy “dinamikus tömb” a JList számára).

- `eloadok`: minden beolvasott előadó/szám
- `kedvencEloadok`: kedvencek listája

### b) `JList<Eloado>`

Ez a képernyőn megjelenő lista komponens.

- `lbZenek` a bal oldali lista, a modellje `eloadok`
- `lbKedvencek` a jobb oldali lista, a modellje `kedvencEloadok`

**Fontos:** A `JList` a modelből rajzol, tehát ha `addElement/remove` történik a modellen, a lista automatikusan frissül.

## 4) `feltolt()` – CSV beolvasás és objektumkészítés

```java
public static void feltolt() {
    try (Scanner scanner = new Scanner(new File("eloadok.csv"), "UTF-8")) {
        scanner.nextLine();

        while (scanner.hasNextLine()) {
            String line = scanner.nextLine();
            String[] parts = line.split(";");
            if (parts.length == 4) {
                String band = parts[0];
                String song = parts[1];
                int year = Integer.parseInt(parts[2]);
                int streams = Integer.parseInt(parts[3]);
                eloadok.addElement(new Eloado(band, song, year, streams));
            }
        }
    } catch (FileNotFoundException e) {
        System.err.println("Hiba: A fájl nem található!");
    } catch (Exception e) {
        System.err.println("Hiba a beolvasás során: " + e.getMessage());
    }
}
```

### Mit vár a program az `eloadok.csv`-től?

- UTF-8 kódolás
- Első sor fejléc (ezért van `scanner.nextLine();`, ami “eldobja”)
- A többi sor formátuma:
    ```
    band;song;year;streams
    ```
    Pontosabban 4 mező `;`-vel elválasztva.

### Mit csinál soronként?

- `line.split(";")` → darabolás
- ha tényleg 4 mező van:
    - `parts[0]` → zenekar/előadó neve
    - `parts[1]` → szám címe
    - `parts[2]` → év (számként)
    - `parts[3]` → stream szám (számként)
- létrehoz egy `Eloado` objektumot és beteszi a bal oldali modellbe: `eloadok.addElement(...)`

### Hibakezelés

- Ha nincs meg a fájl: “A fájl nem található!”
- Egyéb hiba (pl. rossz számformátum): kiírja a hibaüzenetet

**Megjegyzés:** ha a fájlban van szóköz a számok körül (pl. `" 2020"`), akkor `Integer.parseInt` elhasalhat. Ilyenkor jó lenne `.trim()`.

## 5) Ablak létrehozása

```java
public static void createWindow() {
    JFrame frame = new JFrame("Fájlbeolvasás");
    frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
    createUI(frame);
    frame.setSize(1000, 500);
    frame.setLocationRelativeTo(null);
    frame.setVisible(true);
}
```

- `JFrame`: fő ablak
- cím: “Fájlbeolvasás”
- bezáráskor kilép a program (`EXIT_ON_CLOSE`)
- `createUI(frame)` belerakja a komponenseket
- méret: 1000×500
- középre igazítás: `setLocationRelativeTo(null)`
- megjelenítés: `setVisible(true)`

## 6) `createUI()` – teljes felület összerakása (3 oszlop)

```java
JPanel mainPanel = new JPanel(new GridLayout(1, 3, 10, 0));
mainPanel.setBorder(BorderFactory.createEmptyBorder(10, 10, 10, 10));
```

- `GridLayout(1, 3, 10, 0)` → 1 sor, 3 oszlop
- a 3 oszlop közt vízszintesen 10 px “rés”
- körben 10 px margó a panelen

A 3 oszlop:

1. bal panel: “Számok” + lista
2. középen gombok (>, Törlés)
3. jobb panel: “Kedvencek” + lista

## 7) Bal panel (összes szám)

```java
JPanel balPanel = new JPanel(new BorderLayout());

lbZenek.setSelectionMode(ListSelectionModel.SINGLE_SELECTION);
```

- `BorderLayout`:

## 9) “>” gomb működése (kedvencekhez adás)

```java
btnAdd.addActionListener(e -> {
    Eloado kijelolt = lbZenek.getSelectedValue();
    if (kijelolt != null) {
        if (!kedvencEloadok.contains(kijelolt)) {
            kedvencEloadok.addElement(kijelolt);
        } else {
            JOptionPane.showMessageDialog(frame, "Ez a szám már a kedvencek között van!");
        }
    } else {
        JOptionPane.showMessageDialog(frame, "Válassz ki egy számot a bal oldali listából!");
    }
});
```

Lépések:

1. Lekéri a bal listában kijelölt elemet: `lbZenek.getSelectedValue()`
2. Ha nincs kijelölés → felugró ablak: “Válassz ki…”
3. Ha van kijelölt:
    - megnézi, benne van-e már a kedvencek között: `kedvencEloadok.contains(kijelolt)`
    - ha nincs → hozzáadja
    - ha már benne van → felugró: “már a kedvencek között van”

**Nagyon fontos rész:** a `contains(kijelolt)` működése attól függ, hogy az `Eloado` osztály felülírja-e az `equals()`-t (és ideális esetben a `hashCode()`-ot).
- Ha **nincs equals felülírva**, akkor a program csak ugyanazt az objektumpéldányt tekinti “ugyanannak”.
- Itt viszont a kedvencekbe **pont ugyanazt a példányt** teszi át (nem új Eloado-t készít), tehát így is működni fog a duplikáció-védelem ebben a konkrét formában.

## 10) “Törlés” gomb működése (kedvencekből törlés)

```java
btnRemove.addActionListener(e -> {
    int kijeloltIndex = lbKedvencek.getSelectedIndex();
    if (kijeloltIndex != -1) {
        kedvencEloadok.remove(kijeloltIndex);
    } else {
        JOptionPane.showMessageDialog(frame, "Válassz ki egy számot a kedvencek listájából!");
    }
});
```

- Lekéri a jobb listában kijelölt indexet
- Ha nincs kijelölés → `-1` → üzenet
- Ha van → eltávolítja a modellből a megfelelő indexű elemet → a lista frissül

# Amit érdemes még tudni (gyakori kérdések / tipikus buktatók)

## A JList mit fog kiírni az `Eloado` objektumokból?

A `JList<Eloado>` alapból a listaelem `toString()`-jét jeleníti meg.  
Tehát az `Eloado` osztályban jó, ha van ilyen:

```java
@Override
public String toString() {
    return band + " - " + song + " (" + year + ")";
}
```

Ha nincs `toString()`, akkor ilyesmit látsz: `Eloado@5a07e868`.

## Mi van, ha a fájl nem a program mappájában van?

A `new File("eloadok.csv")` relatív útvonal → a **futtatási könyvtárhoz** képest keresi.  
IDE-ben (pl. IntelliJ) ez sokszor a projekt gyökere, de lehet más is.
