## 1) 1 éjszakás edzésterv (csak a tényleg kellő minták)

### Mit kell fejből tudnod reggelig?

1. Kapcsolódás DB-hez `mysqli`-vel + hibaellenőrzés.
2. Alap CRUD:
    - SELECT → while/fetch_assoc → kiírás (lista).
    - INSERT → `$_POST` ellenőrzés → beszúrás.
    - UPDATE → mennyiség növelés/csökkentés.
3. Formkezelés: `method="post"`, input nevek, `isset($_POST['...'])`.
4. Egyszerű védelem: számokra `intval()`, szövegre `mysqli_real_escape_string()`.
5. Oldal automata frissítés: `<meta http-equiv="refresh" content="5">`.
6. Jelszó: `password_hash()` a DB-ben tárolva, ellenőrzés `password_verify()`.
7. Linkelés: lista → `termek.php?id=...` és ott a kiválasztott termék betöltése.

Ennyi. Ha ez megy, az A feladatsor lemegy simán. (Feladatleírások: A és B itt vannak, pont ilyen logika kell. )

### Gyors bemelegítő rutin (kb. 2–3 óra összesen)

- 30 perc: DB létrehozás + helyes séma + admin jelszó felvétele hash-elve.
- 60–90 perc: `index.php`
    - termékek listázása, automata refresh, soronként “készlet +” form,
    - felül új termék form admin jelszóval.
- 30–45 perc: `termekek.php` (linklista) → `termek.php` (részletek + rendelés mínusz készlet, “Elfogyott” logika).
- 20 perc: végigkattintani, hibákat javítani.

## 2) Kész, betanulható minták az A feladathoz

### 2.1. DB script 

```sql
-- db.sql
CREATE DATABASE IF NOT EXISTS raktar CHARACTER SET utf8mb4 COLLATE utf8mb4_hungarian_ci;
USE raktar;

CREATE TABLE IF NOT EXISTS products (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  price DECIMAL(10,2) NOT NULL,
  quantity INT NOT NULL,
  image VARCHAR(255)
);

CREATE TABLE IF NOT EXISTS admin (
  id INT AUTO_INCREMENT PRIMARY KEY,
  password VARCHAR(255) NOT NULL
);
INSERT INTO admin (password) VALUES ('$2y$10$...A_HASH_IDE...');
```

A feladat kéri a hash-elt jelszót (WebAdmin25). Az A leírás is ezt mondja.

## 2.3. `index.php` – lista + “készlet növelés” + “új termék felvétele” admin jelszóval + 5 mp auto-refresh

Feladat szerint nem muszáj táblázat, elég sorokba írni; és 5 mp-enként frissüljön.

```php
<?php
require 'config.php';

$msg = '';

// mennyiség növelés (soronkénti mini form)
if (isset($_POST['add_quantity'])) {
  $id  = intval($_POST['id']);
  $qty = max(0, intval($_POST['add_qty']));
  if ($qty > 0) {
    $conn->query("UPDATE products SET quantity = quantity + $qty WHERE id = $id");
    $msg = 'Mennyiség hozzáadva.';
  }
}

// új termék felvétele admin jelszóval
if (isset($_POST['add_product'])) {
  $name  = $conn->real_escape_string(trim($_POST['name']));
  $price = floatval($_POST['price']);
  $qty   = max(0, intval($_POST['quantity']));
  $pass  = $_POST['admin_pass'] ?? '';

  // admin pass ellenőrzés
  $res = $conn->query("SELECT password FROM admin ORDER BY id ASC LIMIT 1");
  $row = $res ? $res->fetch_assoc() : null;

  if ($row && password_verify($pass, $row['password'])) {
    $conn->query("INSERT INTO products(name, price, quantity) VALUES ('$name', $price, $qty)");
    $msg = 'Új termék rögzítve.';
  } else {
    $msg = 'Hibás admin jelszó.';
  }
}

// termékek lekérése
$products = [];
$res = $conn->query("SELECT id, name, price, quantity FROM products ORDER BY id DESC");
while ($res && $r = $res->fetch_assoc()) { $products[] = $r; }
?>
<!doctype html>
<html lang="hu">
<head>
  <meta charset="utf-8">
  <title>Raktár admin</title>
  <meta http-equiv="refresh" content="5">
  <style>
    body { font-family: system-ui, sans-serif; max-width: 700px; margin: 20px auto; }
    .row { padding: 6px 0; border-bottom: 1px solid #ddd; display:flex; gap:10px; align-items:center; }
    form.inline { display:inline; margin-left: 10px; }
    .msg { background:#f6f6f6; padding:8px; margin-bottom:10px; }
    .header { font-weight:bold; }
  </style>
</head>
<body>
  <h1>Raktár admin</h1>

  <p><a href="termekek.php">Terméklista (linkekkel)</a></p>

  <?php if ($msg): ?><div class="msg"><?= htmlspecialchars($msg) ?></div><?php endif; ?>

  <h2>Új termék felvétele</h2>
  <form method="post">
    <input name="name" placeholder="Név" required>
    <input name="price" type="number" step="0.01" placeholder="Ár" required>
    <input name="quantity" type="number" min="0" placeholder="Mennyiség" required>
    <input name="admin_pass" type="password" placeholder="Admin jelszó (WebAdmin25)" required>
    <button name="add_product">Felvétel</button>

  <h2>Termékek (Név, Ár, Mennyiség)</h2>
  <div class="header row"><div style="width:40%">Név</div><div style="width:20%">Ár</div><div style="width:20%">Mennyiség</div><div>Művelet</div></div>
  <?php foreach ($products as $p): ?>
    <div class="row">
      <div style="width:40%"><?= htmlspecialchars($p['name']) ?></div>
      <div style="width:20%"><?= number_format($p['price'], 2, ',', ' ') ?> Ft</div>
      <div style="width:20%"><?= (int)$p['quantity'] ?></div>
      <form class="inline" method="post">
        <input type="hidden" name="id" value="<?= (int)$p['id'] ?>">
        <input type="number" name="add_qty" min="1" style="width:80px" required>
        <button name="add_quantity">+ hozzáad</button>
      </form>
    </div>
  <?php endforeach; ?>
</body>
</html>
```

## 2.4. `termekek.php` – kattintható névlista

Feladat szerint minden termék neve link, ami a `termek.php`-re visz.

```php
<?php
require 'config.php';
$res = $conn->query("SELECT id, name FROM products ORDER BY name ASC");
$items = [];
while ($res && $r = $res->fetch_assoc()) { $items[] = $r; }
?>
<!doctype html>
<html lang="hu">
<head><meta charset="utf-8"><title>Termékek</title></head>
<body>
  <h1>Termékek</h1>
  <?php foreach ($items as $it): ?>
    <p><a href="termek.php?id=<?= (int)$it['id'] ?>"><?= htmlspecialchars($it['name']) ?></a></p>
  <?php endforeach; ?>
</body>
</html>
```

## 2.5. `termek.php` – kiválasztott termék név+ár + rendelés (készlet csökkentés), “Elfogyott” állapot

Feladat szerinti rendelés-szimuláció: csak akkor csökkenthet, ha van elég darab; ha 0, “Elfogyott”.

```php
<?php
require 'config.php';

$id = intval($_GET['id'] ?? 0);
if ($id <= 0) { die('Hiányzó termék azonosító.'); }

// lekérés
$res = $conn->query("SELECT id, name, price, quantity FROM products WHERE id = $id");
$prod = $res ? $res->fetch_assoc() : null;
if (!$prod) { die('Nincs ilyen termék.'); }

$msg = '';
// rendelés (készlet csökkentése)
if (isset($_POST['order'])) {
  $want = max(0, intval($_POST['order_qty'] ?? 0));
  if ($want <= 0) {
    $msg = 'Adj meg 1 vagy több darabot.';
  } elseif ($prod['quantity'] < $want) {
    $msg = 'Nincs elég készleten.';
  } else {
    $conn->query("UPDATE products SET quantity = quantity - $want WHERE id = $id");
    $prod['quantity'] -= $want; // lokál frissítés
    $msg = 'Rendelés sikeres.';
  }
}
?>
<!doctype html>
<html lang="hu">
<head><meta charset="utf-8"><title><?= htmlspecialchars($prod['name']) ?></title></head>
<body>
  <p><a href="termekek.php">← Vissza a listára</a></p>

  <h1><?= htmlspecialchars($prod['name']) ?></h1>
  <p>Ár: <?= number_format($prod['price'], 2, ',', ' ') ?> Ft</p>

  <?php if ($prod['quantity'] > 0): ?>
    <p>Készleten: <?= (int)$prod['quantity'] ?> db</p>
    <form method="post">
      <input type="number" name="order_qty" min="1" max="<?= (int)$prod['quantity'] ?>" required>
      <button name="order">Rendelés</button>
    </form>
  <?php else: ?>
    <p><strong>Elfogyott</strong></p>
  <?php endif; ?>

  <?php if ($msg): ?><p><?= htmlspecialchars($msg) ?></p><?php endif; ?>
</body>
</html>
```

# 3) Tipikus buktatók, amiket most rögtön javíts

- SQL elgépelések: a te `db.sql`-edben van pár typo: `CREATE DATABES` és `IF NOT EXITS` → ezek miatt nem is fut le. Javítsd a fenti szerint.
- Soha ne vond ki/növeld direkt user-inputtal validálás nélkül. Számokra `intval()/floatval()`, minimum 0.
- Hash ellenőrzésnél mindig `password_verify($plain, $hash)`. A hash-t ne kézzel “kódold”, hanem `password_hash()`-szal generáld és azt illeszd be.
- Auto-refresh: feladathoz elég a meta-tag. Nem kell JS.
- A feladat kimondottan az “alapokkal” kér megoldást; ne bonyolítsd túl (pl. nincs PDO, nincs routing keretrendszer).

# 4) Gyors ellenőrző lista a végén (fél óra)

-  DB létrejött, admin jelszó hash bent.
-  `index.php` 5 mp-enként frissül.
-  Lista: Név, Ár, Mennyiség; soronként mennyiség-hozzáadás működik.
-  Fent új termék form admin jelszóval: hibás jelszóra hiba, jóra INSERT.
-  `termekek.php` linklista → `termek.php?id=...` megnyílik.
-  `termek.php`: név, ár, rendelés csökkent; kevés készletre hiba; 0-nál “Elfogyott”.
-  Nincs notice/warning (hiányzó indexek ellenőrizve).
