## 1. Container – az oldal váza
### Mi a container szerepe?
A `container` a Bootstrap **layout rendszerének alapja**.  
Feladata:
- középre igazítani az oldalt
- biztosítani az egységes oldalszélességet
- megakadályozni, hogy a tartalom „szétfolyjon” nagy kijelzőkön
```html
<div class="container">
  Tartalom
</div>
```
### Mi történik a háttérben?
A Bootstrap:
- breakpointonként eltérő `max-width` értékeket használ
- automatikus oldalsó paddinget ad (`padding-left/right`)
Ez **CSS-ben rengeteg munkát spórol**, különösen reszponzív oldalnál.
---
## 2. Grid rendszer – row és col
### Elvi működés
A Bootstrap **12 oszlopos rácsot** használ.  
Ez nem látszik vizuálisan, de matematikailag minden erre épül.
```html
<div class="row">
  <div class="col-6">Bal</div>
  <div class="col-6">Jobb</div>
</div>
```

### Miért 12?
Mert jól osztható:
- 2, 3, 4, 6 részre is
### Reszponzív működés
```html
<div class="col-sm-12 col-md-6 col-lg-4">
```

Ez azt jelenti:
- mobil: teljes szélesség
- tablet: fél
- desktop: harmad
---
## 3. Tipográfia
### Bootstrap filozófia
A Bootstrap **nem kényszerít**, hanem segít:
- betűméretek
- sorközök
- kontraszt
### Headings osztályok
```html
<p class="h1">Cím</p>
```
Ez:
- vizuálisan `h1`
- szemantikailag `p`
Hasznos, ha:
- SEO miatt nem akarsz sok `h1`-et
- de vizuálisan nagy címet szeretnél
---
### Display osztályok
```html
<p class="display-1">
```
Ez nem klasszikus cím, hanem:
- marketing
- hero section
- landing page elem
---

## 4. Képek
### img-fluid
```html
<img class="img-fluid">
```
Ez:
- `max-width: 100%`
- `height: auto`
---
## 5. Táblázatok
### Alap táblázat

```html
<table class="table">
```
Bootstrap hozzáad:
- paddinget
- kontrasztot
- egységes sortávolságot
### Kiegészítők
- `table-striped`: váltott sorok
- `table-hover`: hover effekt
- `table-bordered`: keretek
Ez tiszta CSS, nincs JS mögötte.
---
## 6. Formok – mély magyarázat
### form-control
```html
<input class="form-control">
```
Ez:
- egységes magasságot ad
- fókusz állapotot kezel
- reszponzív

Bootstrap nélkül:
- minden input másképp néz ki böngészőnként
---
### Checkbox / Radio
```html
<div class="form-check">
```
A Bootstrap:
- összehangolja az inputot és a labelt
- kattinthatóvá teszi a szöveget is
---
### Floating labels
```html
<div class="form-floating">
```
Ez egy **CSS trükk**, amit Bootstrap előre megírt:
- animált címke
- kevesebb hely
- modern UI
---
## 7. Gombok
### btn osztály
```html
<button class="btn btn-primary">
```
A `btn`:
- padding
- border-radius
- hover, active állapotok
### Variánsok
- `btn-success`
- `btn-danger`
- `btn-warning`
- `btn-outline-*`
Ez egy **design system**, nem csak színezés.
---
## 8. Alert
```html
<div class="alert alert-warning">
```
Felhasználás:
- hibaüzenet
- figyelmeztetés
- információ
Bezárható verzió:
- JS-t használ
- eseménykezelés történik

---
## 9. Card
### Miért fontos a card?
A card:
- strukturált tartalomdoboz
- egységes UI

Felépítés:
```html
<div class="card">
  <div class="card-header"></div>
  <div class="card-body"></div>
  <div class="card-footer"></div>
</div>
```
Használat:
- terméklista
- cikkek
- profilok
---
## 10. Navbar
### Navbar filozófia
A navbar egy **összetett komponens**, ami:
- reszponzív
- JS-t használ
- layoutot és logikát is tartalmaz
```html
navbar-expand-lg
```
Ez azt jelenti:
- lg alatt összecsukódik
- hamburger menü jelenik meg
---
## 11. Dropdown
```html
data-bs-toggle="dropdown"
```
Bootstrap 5-ben:
- nincs jQuery
- tiszta JavaScript

Ez:
- eseményfigyelés
- állapotkezelés
---

## 12. Tooltip
```html
data-bs-toggle="tooltip"
```
Tooltip:
- hover esemény
- JS inicializálás kell
- Popper.js számolja a pozíciót
---
## 13. Popover
A Popover a tooltip „nagytestvére”:
- cím
- tartalom
- kattintásra jelenik meg
---
## 14. Toast
A Toast:
- értesítés
- JS vezérelt
- időzített vagy manuális
```js
toast.show()
```

## Bootstrap vs. Sima CSS – Részletes összehasonlítás
## Bootstrap előnyei
### 1. Gyors fejlesztés
- kész osztályok
- nincs nulláról indulás
### 2. Reszponzivitás alapból
- breakpoint rendszer
- grid készen
### 3. Egységes design
- minden gomb, input, alert hasonló
- profi kinézet
---
## Bootstrap hátrányai
### 1. „Bootstrap kinézet”
- felismerhető stílus
- sok oldal hasonló
### 2. Méret
- nem mindig kell minden komponens
---
## Sima CSS előnyei
### 1. Teljes kontroll
- minden pixel a tiéd
- egyedi design
### 2. Könnyebb, kisebb bundle

---
## Sima CSS hátrányai
### 1. Időigényes
- mindent meg kell írni
- reszponzív layout bonyolultabb
### 2. Következetlenség veszélye
- eltérő inputok
- eltérő gombstílusok
