### **Működés:**

1. Ha az első szám (`a`) nagyobb, mint a második (`b`), akkor felcseréli őket.
2. A `for` ciklus visszafelé halad a nagyobb számtól a kisebbig, hogy az első talált prímszám rögtön a legnagyobb legyen.
3. Egy belső `for` ciklus végigellenőrzi, hogy a számnak van-e osztója.
    - Ha **talál osztót**, növeli az `osztókszáma` változót.
    - Ha **nem talál osztót** (vagyis `osztókszáma == 0` marad), akkor az adott szám prímszám, és azt visszaadja.

---

### **Optimalizált változat**

A meglévő megoldás néhány optimalizációval gyorsabbá tehető:

- Nem kell az összes számig ellenőrizni az osztókat, csak a **négyzetgyökéig**.
- A `range(b, a-1, -1)` iterációba beépíthető, hogy **csak páratlan számokat** nézzen (kivéve a 2-est).

```python
import math

def lnp(a, b):
    if a > b:
        a, b = b, a  # Egyszerűbb csere

    def prím_e(n):
        if n < 2:
            return False
        if n == 2 or n == 3:
            return True
        if n % 2 == 0:
            return False
        for d in range(3, int(math.sqrt(n)) + 1, 2):
            if n % d == 0:
                return False
        return True

    for i in range(b, a - 1, -1):
        if prím_e(i):
            return i

    return None  # Ha nincs prím az intervallumban
```

---

### **Miben jobb ez a verzió?**

**Hatékonyabb:**
- Csak a négyzetgyökéig ellenőrzi az osztókat.
- A páros számokat kizárja az ellenőrzésből (kivéve a 2-t).
**Tisztább szerkezet:**
- A prímellenőrzést külön függvény végzi (`prím_e()`), így könnyebben olvasható és újrafelhasználható.

**Helyesen kezeli a szélsőséges eseteket:**

- Ha nincs prím az adott tartományban, `None`-t ad vissza.

---

### **Példa futtatások**

```python
print(lnp(2, 10))   # → 7
print(lnp(10, 2))   # → 7 (kezeli a fordított sorrendet)
print(lnp(20, 30))  # → 29
print(lnp(90, 100)) # → 97
print(lnp(8, 10))   # → None (nincs prím a tartományban)
```

🔹 Ez a verzió gyorsabb és hatékonyabb nagyobb számok esetén is! 🚀
```python
Feladat:
Egy adott intervallumon belül akarjuk megkeresni a legnagyobb prímszámot.
Pl.: 
lnp(2,10) --> 7 (mert 2 és 10 között 7 a legnagyobb prím)
    - Ha a nagyobb szám van hamarabb, úgy tudja értelmezni
Tippek:
    for ciklus, range(), if, %, ==, belső for ciklus
'''
def lnp(a,b):
    if a > b: #Ha a nagyobb van elől, akkor felcseréljük
        c = a
        a = b
        b = c
    for i in range(b,a-1,-1): #-1: visszafele lépkedjen a nagyobbtól
        osztókszáma = 0
        for d in range(2,int(i/2)+1):
            if i % d == 0: #Ha talál egy osztót
                osztókszáma += 1 #azt számolja fel
#Ha nem talált osztókat az adott számhoz, akkor az a megoldás
#(ennél már csak kisebbeket találhatna, mert visszafele haladunk)
        if osztókszáma == 0:
            return i
```
