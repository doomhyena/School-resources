# IT Redundancia

## Mi az a redundancia?

Az IT redundancia tervezései és üzemeltetési elv, 
amelynek célja, hogy  az egy rendszer meghibásodása esetén is működő képes maradjon vagy minimális szolgáltatás kiesés történjen

## A redundamcia célja

1. Maga rendelkezésre alias (High Availablity - HA)
2. Adatvesztés minimalizálása 
3. Karbantartás aéatto működés fenntartása
4. Kockázat csökkentés

## Alapfogalmak

1. Single Point of Failure (SPF): Olyan rendszerkomponens amelynek a meghibásodása az egész szolgáltatás leállását erendényezi. A rendundancia célja ezeknek a megszűntetése.
2. Recovery Time Object (RTO): Mennyi dő alatt kell helyreállnia a rendszernek.
3. Recpvery Point Objective (RPO): Mennyi adat vesztése elfogadható

## Redundancia Típusa

1. Hardver redundamcia
    1. A fizikai eszközök megkettőzése vagy megtöbszörösözése
    2. Példák:
        1. Kettős tápegysés
        2. RAID-es háttértárak
        3. Rendundás hálózati kártyák
        4. Több szerver azonos szerepkörökben
2. Szoftveres redundamcia
    1. Logikai vagy alkalmazásszintű megoldások
    2. Példák:
        1. Cluster megoldások
        2. Load balancer
3. Hálózati redundamcia
    1. A hálózat elérhetőségének biztosítása
    2. Példák:
        1. Több útvonal
        2. Redundás router és switcj
        3. Több internet és szolgáltatás
        4. VLAN és routing rendundancia
4. Adat redundancia
    1. Az adatok elérhetősége és védeleme
    2. Példák:
        1. RAID (RAID 0, 1, 5, 6, 10 vagy 1+0)
        2. Adatreplikáció (szinkron/aszinkron)
        3. Backup rendszerek
        4. Geo-redundancia

## Georedundancia architecturáia

1. Active-Passive
    1. Egy aktív rendszer
    2. Egy készenléti tartalék
    3. Hiba esetén átváltás
    4. Előnye: Egyszerű
    5. Hátránya: kihasználatlan errőfrorrások
2. Active-Active
    1. Több rendszer egyszerre aktív
    2. Terhelés megoldás
    3. Előnye: Magasabb teljesítmény, jobb kihasználtság
    4. Hátrány: Bonyolultabb tervezés

## Redundancia vs Backup vs DR

| Fogalom           | Cél                |
| ----------------- | ------------------ |
| Redundancia       | Folyamatosműködés  |
| Backup            | Aadatvisszaállítűs |
| Disaster Recovery | Katasztrófa tűrés  |
