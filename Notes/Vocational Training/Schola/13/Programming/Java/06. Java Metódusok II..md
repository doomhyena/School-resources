**Saját metódusok, avagy ne ismételd önmagad**

A programozás során gyakran előfordul, hogy ugyanazt a kódot többször is felhasználjuk. Ilyenkor hatékonyabb megoldás, ha az ismétlődő részeket metódusokba szervezzük. A Java nyelvben a metódusok segítségével logikailag elkülöníthető részfeladatokat tudunk definiálni, és ezeket a kódunk bármely pontjáról hívhatjuk.

### Metódusok típusai

A metódusok két alapvető csoportba sorolhatók:

1. **Feladatvégrehajtás visszatérési érték nélkül** (pl. kiírás a képernyőre).
2. **Feladatvégrehajtás visszatérési értékkel** (pl. matematikai számítás, ahol az eredményt visszaadjuk).

### Metódusok szerkezete

A metódusok két fő részből állnak:

- **Fejrész**: Megadja a metódus nevét, a visszatérési típust, a paramétereket, és esetleges módosítókat (`public static` stb.).
- **Törzs**: Tartalmazza a metódus végrehajtandó utasításait.

Példa egy egyszerű metódusra, amely vonalat rajzol:

```java
public static void vonal() {
  System.out.println("-----");
}
```

Példa egy metódusra, amely visszatér egy véletlen számmal az adott intervallumból:

```java
public static int veletlen() {
  return (int)(Math.random() * 41) - 10;
}
```

### Metódus hívások

A metódusok hívásához a metódus nevét használjuk, és megadjuk a szükséges paramétereket:

```java
vonal(); // Kiír egy vonalat
int szam = veletlen(); // Véletlen szám generálása
```

### Paraméterek és visszatérési érték

Metódusokat úgy is definiálhatunk, hogy külső adatokat kapjanak, és ezekkel dolgozzanak:

```java
public static int atlag(int szam1, int szam2) {
  return (szam1 + szam2) / 2;
}
```

Hívása:

```java
int eredmeny = atlag(10, 20);
```

### Metódusok túltelhelése (Overloading)

Azonos nevű metódusokat is definiálhatunk, ha azok paraméterlistája eltér:

```java
public static void vonal() {
  System.out.println("-----");
}

public static void vonal(int hossz) {
  for (int i = 0; i < hossz; i++) {
    System.out.print("-");
  }
  System.out.println();
}
```

### Referencia típusok átadása metódusnak

Amikor egy metódusnak egy tömböt adunk át, akkor valójában annak referenciáját kapja meg, így a metódusban történő módosítás az eredeti tömbre is hat:

```java
public static void novel(int[] tomb) {
  for (int i = 0; i < tomb.length; i++) {
    tomb[i]++;
  }
}
```

Ha viszont meg akarjuk őrizni az eredeti adatokat, akkor a `clone()` metódust kell használni:

```java
int[] ujTomb = tomb.clone();
novel(ujTomb);
```

### Összegzés

A metódusok segítenek a kód olvashatóságának javításában, az ismétlések elkerülésében, és az útírahasznosítható kód írásában. Az alap metódusok használata mellett a túltelhelés, a visszatérési értékek kezelése, valamint a referencia típusok átadása is fontos szerepet játszik a hatékony programozásban.
